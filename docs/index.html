Databases for text analysis

DocTable is a Python package for parsing, storing, and accessing text documents and models for large scale text analysis.

Here's the scenario:

You're starting a new text analysis project. You need to take raw text and convert it into a format which can be used for your analysis. The data is big. It may require preprocessing, tokenization, or parsetree extraction, and all needs to be stored and accessed efficiently in a single file. Databases may be your best option, but SQL is a pain. In this case, DocTable could be a good way to work with your data.

AND/OR: Your text analysis algorithms produce large models, and you may need to generate several of them with different parameter configurations. DocTable is good for that too. DocTable2 is equipped with custom column types for storing pickled python objects, including ones optimized for parsed document objects.

What's in it:

DocTable is composed of two Python classes: DocTable2 for creating and manipulating database tables, and DocParser for tokenizing and extracting parsetrees. See the documenation below for examples and references.
