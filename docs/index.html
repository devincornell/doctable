Databases for text analysis

doctable is a python package for designing and manipulating database tables through an object-oriented interface.

This package allows you to interface with database tables as objects, and provides functionality especially aimed at .

  1. Create a schema object using the @doctable.schema decorator. In addition to providing the schema definition, this class operates much like a dataclass so you may add custom methods for working with retrieved or inserted database rows.
  2. Define the interface to your data model by creating a class that inherits from DocTable. The base class provides methods for inserting, retrieving, and changing database rows.
  3. Develop a text parsing pipeline using a ParsePipeline class. This class can distribute your text parsing across multiple processes, and can be used to create ParseTree objects to be stored in your database.
  4. Merge your parsed text with relevant document metadata and use your 
    DocTable
    interface to store and retrieve data from the database.

Other Utilities

doctable includes a number of other tools for text analysis and database management.

  • Timer: used to time function calls and log your scripts at different stages.
  • FSStore: interact with data rows directly on the filesystem. Great for parallel processing.
  • TempFolder: create temporary folder that will be deleted upon garbage collection.
